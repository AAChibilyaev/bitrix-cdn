# Резервное копирование и восстановление

В этом документе описаны шаги по созданию и восстановлению резервных копий данных MinIO-бакета.

---

## 1. Настройка клиента MinIO (mc)

1. Установите MinIO Client (mc), если он ещё не установлен:  
   https://min.io/docs/minio/linux/reference/minio-mc.html#install-mc  
2. Создайте временный или постоянный alias для доступа к вашему серверу MinIO:

   ```bash
   # Переменные (если нужно):
   #   MINIO_ENDPOINT — URL MinIO (по умолчанию http://localhost:9000)
   #   MINIO_ROOT_USER — из .env
   #   MINIO_ROOT_PASSWORD — из .env

   mc alias set local "${MINIO_ENDPOINT:-http://localhost:9000}" \
     "${MINIO_ROOT_USER}" "${MINIO_ROOT_PASSWORD}" --api S3v4
   ```

3. Проверьте доступ к бакету:

   ```bash
   mc ls local/"${MINIO_BUCKET_NAME}"
   ```

---

## 2. Резервное копирование

Для создания резервной копии выполните скрипт:

```bash
./scripts/backup.sh
```

Скрипт автоматически:

- Загружает переменные из `.env`
- Создаёт директорию `backups/` (если нужно)
- Формирует подпапку с временным штампом `YYYY-MM-DD_HH-MM-SS`
- Делает `mc mirror` бакета в локальную папку
- Удаляет временный alias

Пример вывода:

```
=== Начало резервного копирования бакета: my-cdn-bucket ===
...
Резервная копия создана: backups/my-cdn-bucket-2025-04-18_18-50-00 (размер: 1.2G)
=== Резервное копирование завершено ===
```

### Автоматизация (cron)

Добавьте задание в crontab, например:

```cron
0 3 * * * cd /path/to/project && ./scripts/backup.sh >> /var/log/backup.log 2>&1
```

Это будет запускать бэкап каждый день в 03:00.

---

## 3. Восстановление

Для восстановления данных из резервной копии используйте скрипт:

```bash
# Укажите путь к директории с резервной копией:
./scripts/restore.sh backups/my-cdn-bucket-2025-04-18_18-50-00
```

Скрипт:

- Загружает переменные из `.env`
- Проверяет существование указанной директории
- Делает `mc mirror --overwrite` из локальной папки в бакет
- Удаляет временный alias

Пример вывода:

```
=== Начало восстановления бакета: my-cdn-bucket ===
Запуск mc mirror: backups/my-cdn-bucket-2025-04-18_18-50-00/ -> local/my-cdn-bucket/
=== Восстановление завершено ===
```

---  
Все скрипты расположены в каталоге `scripts/` и снабжены детальными комментариями на русском языке.
